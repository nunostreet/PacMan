## Description

A fully playable Pac-Man clone written in Python using pygame. The game features procedurally generated mazes (via an external A-Maze-ing package), four ghost enemies with chase/scatter AI, a persistent highscore system, a complete UI flow (main menu, pause, game over, victory), and cheat mode features (invincibility, freeze, etc.). The game spans 10 progressively larger levels; the player keeps their score and remaining lives between levels.

## How to play

### Controls

| Key | Action |
|-----|--------|
| `W` / `A` / `S` / `D` | Move up / left / down / right |
| `Esc` | Pause / resume |

Eat all the dots to complete the level. Avoid ghosts — unless you've just eaten a super-pacgum (large dot), which makes all ghosts edible for a few seconds. Surviving all 10 levels wins the game.

### Cheat mode

These keys work during gameplay. Using any cheat disables highscore saving for that run.

| Key | Effect |
|-----|--------|
| `I` | Toggle invincibility (ghosts can't kill you) |
| `F` | Freeze / unfreeze all ghosts |
| `L` | Skip to the next level |
| `B` | Go back to the previous level |
| `M` | Add one life (max 7) |

## Highscore

Scores are stored in a JSON file (default: `highscores.json`, configurable via `highscore_filename` in the config). The system keeps the top 10 scores sorted in descending order.

- **When it's saved**: after a game ends (win or lose), the player is prompted to enter their name. The score is only saved if no cheat was used during that run.
- **Player names**: 1–10 characters, alphanumeric and spaces only; enforced in the input handler.
- **File format**: a JSON array of `{"name": "...", "score": N}` objects.
- **Robustness**: if the file is missing or corrupted, the game starts with an empty list and creates a clean file on the next save.
- **Display**: visible from the main menu under "View Highscores".

We chose a flat JSON file because it is easy to read and follow.

## Maze Generation

Mazes are generated by the assigned A-Maze-ing package (`mazegenerator`), bundled as a `.whl` in the repository root and installed via `make install`.

The integration lives in `engine/maze.py`:

1. `MazeGenerator(size=(width, height), perfect=False, seed=seed)` is called with `PERFECT=False` to produce Pac-Man-compatible corridors (multiple paths between cells).
2. Any stdout output from the package is captured with `contextlib.redirect_stdout` and logged as a warning to keep the terminal clean.
3. If generation fails for any reason, the exception is caught, logged, and the function returns `False` — the engine handles this gracefully.
4. From the raw bitmask grid (`mg.maze`), we build an **adjacency list** (`neighbors`) in one pass: for each cell, we check the four wall bits (N=1, E=2, S=4, W=8) and record which neighbours are reachable. This gives O(1) wall checks at runtime.
5. Pacgums are placed randomly across open corridors; super-pacgums are always placed in the four maze corners.

The first level uses a fixed seed (configurable, default `42`) for a reproducible starting experience. All subsequent levels use a random seed.

## Implementation

### Movement and timing

Entity speed is decoupled from frame rate via a **speed accumulator**. Each frame, `dt` (seconds since last frame) is added to `_pacman_timer` and `_ghost_timer`. When an accumulator reaches its interval (0.25 s for Pac-Man, 0.30 s in chase, 0.45 s in flee for ghosts), the entity moves one cell and the timer resets to 0.

The `GameSnapshot` exposes `move_alpha = _pacman_timer / _pacman_interval`, which is the sub-cell progress between 0.0 (just moved) and ~1.0 (about to move again). The UI uses it to interpolate the sprite position between the previous and current grid cell each frame, so movement looks smooth instead of jumping cell to cell.

### Ghost AI

Each ghost uses **greedy Manhattan distance** to pick the next cell:
- **Chase mode**: picks the open neighbour closest to Pac-Man.
- **Flee mode**: picks the open neighbour furthest from Pac-Man, with 30% random noise to make ghosts catchable.
- **Scatter mode**: targets the ghost's own spawn corner instead of Pac-Man.

A **no-reverse rule** prevents ghosts from immediately turning back — the last visited cell is excluded from the candidate list unless it is the only option. This eliminates oscillation in straight corridors.

Scatter and chase phases alternate on a fixed timer (`[7s, 20s, 7s, 20s, 5s, 20s, 5s]`). When any ghost is edible, the scatter/chase timer is paused.

### Collision

Collision is checked only when Pac-Man's `move_alpha >= 0.5` — i.e., once he is visually halfway into a cell. This prevents instant ghost hits at cell boundaries and keeps the feel consistent with the original game.

### Cheat mode

Cheats are toggled during gameplay with keyboard shortcuts (see Instructions). Any cheat use sets `cheat_used = True` in the snapshot, which prevents the final score from being saved to the highscores.

## General Software Architecture

```
pac-man.py              — entry point; argument parsing and game launch
contracts.py            — shared data types (Direction, GameStatus, GameSnapshot,
                          GameConfig, GhostState, PacmanGameProtocol)
config/
  parser.py             — command-line + JSON config parsing, validation, defaults
engine/
  maze.py               — MazeGenerator wrapper, adjacency list, pacgum placement
  player.py             — Pac-Man position and movement
  ghost.py              — Ghost AI: greedy chase/flee, scatter, no-reverse, timers
  game.py               — Master game loop: tick(), collision, level management,
                          cheat mode, scatter/chase cycle
interface/
  app.py                — pygame init, screen transitions, main loop
  game_screen.py        — maze renderer, sprite interpolation, HUD
  main_menu.py          — navigation menu
  pause_menu.py         — pause overlay
  game_over.py          — game over screen + name input
  victory.py            — victory screen (inherits GameOver)
  highscores.py         — highscore draw/load/save
  instructions.py       — controls reference screen
  option_menu.py        — base keyboard-navigable menu
  interface_screen.py   — abstract base class for all screens
```

**Key design principle**: the engine (`engine/`) has zero pygame dependency and is fully testable without a display. The UI reads game state exclusively through `GameSnapshot` (a plain dataclass). `PacmanGameProtocol` (a `typing.Protocol`) defines the engine's public surface, avoiding circular imports between the two layers.
